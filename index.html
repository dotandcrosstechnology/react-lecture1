üìö What is React? 13:04

React is a popular JavaScript library for building user interfaces, particularly single-page applications where you need a fast, interactive user experience. Developed and maintained by Facebook (Meta), React has become one of the most widely used front-end libraries in the world.

Key Features of React:

  • Component-Based Architecture: Build encapsulated components that manage their own state
  • Virtual DOM: Efficient rendering through a lightweight copy of the actual DOM
  • Declarative: Design simple views for each state in your application
  • Learn Once, Write Anywhere: Develop new features without rewriting existing code
üí° Did you know? React was first deployed on Facebook's News Feed in 2011 and on Instagram in 2012.
Simple React Example 
import React from 'react';

function Welcome() {
  return <h1>Hello, React!</h1>;
}

export default Welcome;

üé® What is JSX? 03:41

JSX (JavaScript XML) is a syntax extension for JavaScript that allows you to write HTML-like code within JavaScript. It makes it easier to write and understand the structure of your components.

Why JSX?

  • Makes code more readable and easier to understand
  • Prevents injection attacks by escaping embedded values
  • Allows you to use the full power of JavaScript in your markup
  • Faster than regular JavaScript because it performs optimizations
JSX Example 
// JSX allows you to write HTML-like syntax
const element = <h1>Hello, JSX!</h1>;

// JSX with JavaScript expressions
const name = 'Anas';
const greeting = <h1>Hello, {name}!</h1>;

// JSX with attributes
const link = <a href="https://reactjs.org">Learn React</a>;
‚ö†Ô∏è Important: JSX is not valid JavaScript. It needs to be transpiled to regular JavaScript using tools like Babel.

üéØ Our 1st Component 08:46

Components are the building blocks of any React application. Let's create our first custom component!

What is a Component?

A component is a reusable piece of UI that can be composed together to create complex interfaces. Think of components as JavaScript functions that return HTML elements.

Creating Your First Component 
// Greeting.js
function Greeting() {
  return (
    <div>
      <h2>Hello, Welcome to React!</h2>
      <p>This is my first custom component.</p>
    </div>
  );
}

export default Greeting;
Using the Component in App.js 
import Greeting from './Greeting';

function App() {
  return (
    <div className="App">
      <Greeting />
    </div>
  );
}

export default App;
üí° Best Practice: Component names should always start with a capital letter to distinguish them from regular HTML elements.

üì¶ Import-Export 04:45

Import and export statements allow you to split your code into multiple files and reuse components across your application.

Export Types:

Default Export 
// Button.js
function Button() {
  return <button>Click Me</button>;
}

export default Button;

// Import in another file
import Button from './Button';
Named Export 
// utils.js
export function add(a, b) {
  return a + b;
}

export function subtract(a, b) {
  return a - b;
}

// Import in another file
import { add, subtract } from './utils';
Mixed Exports 
// helpers.js
export function formatDate(date) {
  return date.toLocaleDateString();
}

export function formatTime(date) {
  return date.toLocaleTimeString();
}

function MainHelper() {
  return "Main Helper";
}

export default MainHelper;

// Import
import MainHelper, { formatDate, formatTime } from './helpers';

üìù Writing Markup in JSX 05:52

JSX lets you write HTML-like markup inside JavaScript, but there are some important rules to follow.

JSX Rules:

  • Return a single root element: Wrap multiple elements in a parent tag
  • Close all tags: Even self-closing tags like <img /> must have a closing slash
  • Use camelCase: Most HTML and SVG attributes are written in camelCase
  • className instead of class: Use className for CSS classes
Correct JSX Markup 
function Profile() {
  return (
    <div>
      <img 
        src="avatar.jpg" 
        alt="Profile" 
        className="profile-img"
      />
      <h1>John Doe</h1>
      <p>Web Developer</p>
      <label htmlFor="email">Email:</label>
      <input 
        id="email" 
        type="email" 
        placeholder="Enter email"
      />
    </div>
  );
}
‚ö†Ô∏è Common Mistakes:
  • Using "class" instead of "className"
  • Forgetting to close self-closing tags
  • Returning multiple elements without a wrapper
  • Using "for" instead of "htmlFor" in labels

üß© React Fragment 01:50

React Fragments let you group multiple elements without adding extra nodes to the DOM.

Why Use Fragments?

Sometimes you need to return multiple elements from a component, but you don't want to add an unnecessary div wrapper that could affect your CSS or DOM structure.

Using React.Fragment 
import React from 'react';

function List() {
  return (
    <React.Fragment>
      <h2>Shopping List</h2>
      <ul>
        <li>Apples</li>
        <li>Bananas</li>
        <li>Oranges</li>
      </ul>
    </React.Fragment>
  );
}
Short Syntax (Recommended) 
function List() {
  return (
    <>
      <h2>Shopping List</h2>
      <ul>
        <li>Apples</li>
        <li>Bananas</li>
        <li>Oranges</li>
      </ul>
    </>
  );
}
üí° Tip: The short syntax <>...</> is preferred unless you need to pass keys or attributes, in which case use <React.Fragment>.

üî¢ JSX with Curly Braces 02:27

Curly braces {} let you embed JavaScript expressions directly in your JSX markup.

Using JavaScript Expressions in JSX 
function UserProfile() {
  const name = "Anas Yousaf";
  const age = 25;
  const skills = ["React", "JavaScript", "Node.js"];
  
  return (
    <div>
      <h1>{name}</h1>
      <p>Age: {age}</p>
      <p>Birth Year: {2024 - age}</p>
      <p>Skills: {skills.join(", ")}</p>
      <p>{age >= 18 ? "Adult" : "Minor"}</p>
    </div>
  );
}
Using Objects and Functions 
function ProductCard() {
  const product = {
    name: "Laptop",
    price: 999,
    inStock: true
  };
  
  const formatPrice = (price) => `$${price.toFixed(2)}`;
  
  return (
    <div>
      <h2>{product.name}</h2>
      <p>Price: {formatPrice(product.price)}</p>
      <p>Status: {product.inStock ? "Available" : "Out of Stock"}</p>
      <img 
        src="laptop.jpg" 
        alt={product.name}
        style={{
          width: "200px",
          borderRadius: "10px"
        }}
      />
    </div>
  );
}

What You Can Put in Curly Braces:

  • Variables and constants
  • Expressions (mathematical operations, string concatenation)
  • Function calls
  • Ternary operators
  • Array methods (map, filter, etc.)

Component Composition Example 
// Card.js - Reusable component
function Card({ children, title }) {
  return (
    <div className="card">
      <h3>{title}</h3>
      <div className="card-content">
        {children}
      </div>
    </div>

Best Practices for Component Structure:

  • Single Responsibility: Each component should do one thing well
  • Reusability: Create generic components that can be used in multiple places
  • Composition: Build complex UIs by combining simple components
  • Clear Naming: Use descriptive names that indicate the component's purpose
üí° Pro Tip: Keep components small and focused. If a component is doing too much, break it into smaller components.

üé® Style Components 05:49

There are multiple ways to style React components. Let's explore the most common methods.

1. Inline Styles:

Inline Styling with Objects 
function StyledButton() {
  const buttonStyle = {
    backgroundColor: '#667eea',
    color: 'white',
    padding: '10px 20px',
    border: 'none',
    borderRadius: '5px',
    cursor: 'pointer',
    fontSize: '16px'
  };
  
  return (
    <button style={buttonStyle}>
      Click Me
    </button>
  );
}

// Or directly inline
function DirectInline() {
  return (
    <div style={{ 
      backgroundColor: '#f7fafc', 
      padding: '20px',
      margin: '10px' 
    }}>
      Content
    </div>
  );
}

2. CSS Stylesheets:

Component.js 
import './Button.css';

function Button() {

3. CSS Modules:

Component with CSS Module 
// Card.js
import styles from './Card.module.css';

function Card() {
  return (
    <div className={styles.card}>
      <h2 className={styles.title}>Card Title</h2>

}
Card.module.css

  color: #667eea;
  margin-bottom: 10px;

4. Conditional Styling:

Dynamic Class Names 
function Alert({ type, message }) {
  // Conditional className
  const alertClass = type === 'error' ? 'alert-error' : 'alert-success';
  
  return (
    <div className={`alert ${alertClass}`}>
      {message}
    </div>
  );
}

// Template literals for multiple classes
function Button({ primary, large }) {
  return (
    <button 
      className={`btn ${primary ? 'btn-primary' : 'btn-secondary'} ${large ? 'btn-large' : ''}`}
    >
      Click Me
    </button>
  );
}

Styling Comparison:

When to use each method:
  • Inline Styles: Dynamic styles that change based on props or state
  • CSS Stylesheets: Global styles and simple component styling
  • CSS Modules: Component-scoped styles to avoid naming conflicts
  • Styled Components/Emotion: Advanced styling with JavaScript (requires additional libraries)
